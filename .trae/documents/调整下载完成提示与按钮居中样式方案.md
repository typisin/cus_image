## 目标
- Toast 仅在“文件成功写入本地”后展示
- `Reupload` 与 `Download` 两个按钮文字居中，对齐图标并缩短右侧间距

## 下载完成的准确判定
- 使用 `File System Access API`（`showSaveFilePicker` + `createWritable` + `write` + `close`）：
  - 当 `await writable.close()` 解决后，表示文件已成功写入本地，再触发 Toast
  - 浏览器支持：Chromium系（Chrome/Edge）支持良好；Safari/Firefox不完全支持
- 兼容策略：
  - 首选 `File System Access API`；若不支持则回退为现有的 `a.download` 方案
  - 在回退方案中不提前显示成功，只显示轻量提示“Preparing download...”或维持静默（可选），不误导为完成

## 具体实现（JS）
- 在 `cutout.js` 中新增能力检测与保存流程：
  - 检测 `window.showSaveFilePicker`
  - 若支持：
    - 取 `this.cutoutBlobUrl`（若无则先 `fetch(this.cutoutImage)` 获取 Blob）
    - 调用 `showSaveFilePicker({ suggestedName: 'ai-cutout-<timestamp>.png', types: [{ description: 'PNG Image', accept: { 'image/png': ['.png'] } }] })`
    - `const handle = await showSaveFilePicker(...); const writable = await handle.createWritable(); await writable.write(blob); await writable.close();`
    - 完成后 `showToast('Download complete')`
  - 不支持：
    - 保留现有下载逻辑，但不再在点击瞬间显示“完成”，避免误判

## 按钮居中与间距（CSS）
- 统一 `.btn` 为图标+文字居中布局：
  - `display: inline-flex; align-items: center; justify-content: center; gap: 0.375rem;`
  - 将 `padding` 调整为左右更紧凑：如 `padding: 0.5rem 0.875rem;`（原右侧过宽）
- 两个按钮作用域：
  - 由于 `cutout.css` 中存在两个 `.btn` 定义，计划合并为单一定义，避免覆盖混乱
  - 针对结果区域的按钮（含图标），如需更细粒度，可用 `.results-actions .btn` 控制 `gap` 与 `min-width`，保证文字居中
- 适配 `.btn-primary` 与 `.btn-secondary`：
  - 保持颜色与边框样式不变，只继承新的居中与内边距

## Toast 样式保持
- 保持白蓝配色的居中弹层：
  - `background: var(--white)`、`border: 1px solid var(--primary-blue)`、`color: var(--primary-blue)`
  - 位置：`top: 50%; left: 50%; transform: translate(-50%, -50%)`
  - 动画：使用 `var(--transition-fast)`，避免延后显示
